Команды Гит для локальной работы

Команда Что делает
git init инициализация репозитория в текущем каталоге
git clone git://host.org/project.git клонирование удаленного репозитория
git add new_filename добавление нового файла
git add . Вносит в индекс все изменния, включая обновления и новые файлы
git add src/ Добавление каталога
git rm filename удаление файла/директории из рабочей директории
git rm -r src/ Удаление каталога рекурсивно (со всем содержимым)
git rm –cached –ignore-unmatch file.ext удаляем файл только из репозитория, и физически файл сохранится на диске
git rm –cached –ignore-unmatch –r src/ удаляем папку только из репозитория, рекурсивно (с вложенными папками и файлами и физически файл сохранится на диске
git mv from to перемещение файла/директории
git status состояние рабочей директории
git commit –a –m “comment” Одновременное добавление и фиксация изменений
git commit -a –amend внесение исправлений в последний коммит
git commit –amend Позволит добавить исправление в комментарии последнего коммита, отправив в текстовый редактор
git blame filename кто и что менял в файле filename
git log история изменений
git log –graph –pretty=oneline графический лог изменений
git log –oneline –decorate –graph –all Красивый лог
git log –pretty=format:”%h %s” –graph Графический лог изменений с форматированием
git reset Сброс индекса если есть
git reset file.txt Сброс индекса для указанного файла, есди он внесен в индекс
git diff REV_ID1 REV_ID2 Изменения между ревизией REV_ID1 и REV_ID2
git show REV_ID:filename Показать конкретный файл filename из ревизии REV_ID
git reset –hard Вернутся к предыдущему состоянию рабочей директории
git add .
git commit –amend		 Если добавить файлы в индекс то они добавятся вместе с предложением исправить комментарий
git branch Список всех веток
git branch branch_name Создание ветки branch_name
git branch -d branch_name удаление ветки branch_name
git checkout branch_name переключение на ветку branch_name
git checkout branch_name1
git merge branch_name2		 Слияние веток branch_name1 и branch_name2 сливается в ту ветку в которой сейчас стоим
git stash Затаить файлы
git stash list Посмотреть список затаенных файлов
git stash apply stash@{2} Применить определенный тайник, при этом он останется в памяти
git stash drop stash@2 Удалить определенный тайник
git stash pop Применить и сразу удалить тайник
git stash apply Применить последний
git tag tag_name Помечаем текущую ревизию тегом tag_name легкие без дополнительной иформации
git tag Просмотр всех тэгов (меток)
git tag –a “version 1.0” –m “release1.0” Установка тэга (метки) Аннотированные с доп. данными

Команды Гит для работы c удаленным репозиторием

Команда Что делает
git push –u origin name-branch Отправляем изменения ветки в удаленный репозиторий Ключ –u привязывает ветку в лок. рпз с удаленным
git fetch Получаем последние обновления из origin (удаленного репозитория) без слияния с текущем состоянием
git pull origin master Получаем последние обновления из origin репозитория) со слиянием с текущем состоянием)
git branch -u origin/master
git pull 		Установка ветки upstream получение обновлений
git push Вносим изменения в origin
git push –tags Отправка меток в удаленное хранилище
